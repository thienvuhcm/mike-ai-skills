# Project Lombok — Eliminate Java Boilerplate with Compile-time Code Generation

Write, review, debug, and refactor Java that uses Project Lombok by applying the official Lombok feature documentation. Lombok is a **compile-time annotation processor**: small annotations on your classes/fields/methods cause Lombok to inject getters, setters, constructors, `equals`/`hashCode`/`toString`, builders, loggers, and more directly into the bytecode — so the result is exactly as if you had hand-written it (no reflection, no runtime cost). This skill teaches the idiomatic usage of every feature, what each one **generates**, and the pitfalls that bite in real projects.

**Target version:** Lombok **1.18.x** (current stable **1.18.46**, 2026-04). `groupId:artifactId` = `org.projectlombok:lombok`. Newer features carry a **Since** note (e.g. `@Locked` 1.18.32, `@StandardException` 1.18.22, `@Jacksonized` 1.18.16, `@SuperBuilder` 1.18.2) — see the reference's version table.

**What is covered in this Skill?** (organized by the reference's 8 chapters)

- **Setup & model** (Ch 1): compile-only dependency + annotation processor (Maven/Gradle/IDE); how Lombok rewrites the AST and skips members you wrote yourself
- **Locals, resources & exceptions** (Ch 2): `val`/`var`, `@NonNull` (null-check insertion), `@Cleanup` (resource closing), `@SneakyThrows`
- **Accessors & object methods** (Ch 3): `@Getter`/`@Setter` (+ `AccessLevel`), `@ToString`, `@EqualsAndHashCode` (+ `callSuper`, `@Include`/`@Exclude`), `@Getter(lazy=true)`
- **Constructors & aggregates** (Ch 4): `@NoArgsConstructor`/`@RequiredArgsConstructor`/`@AllArgsConstructor` (+ `staticName`, `access`), `@Data`, `@Value` (immutable), `@With` (copy-on-write)
- **Builders** (Ch 5): `@Builder` (+ `toBuilder`, `builderMethodName`…), `@Singular`, `@Builder.Default`/`@Builder.ObtainVia`, `@SuperBuilder`, `@Jacksonized`
- **Concurrency & logging** (Ch 6): `@Synchronized`, `@Locked` (+ `.Read`/`.Write`, virtual-thread friendly), the `@Log` family (`@Slf4j`/`@Log4j2`/`@CommonsLog`/`@JBossLog`/`@Flogger`/`@Log`/`@Log4j`/`@XSlf4j`/`@CustomLog`)

- **Configuration** (Ch 8): the `lombok.config` system (hierarchy, `config.stopBubbling`, `flagUsage`, null annotations, `addLombokGeneratedAnnotation`) and combining with MapStruct via `lombok-mapstruct-binding`

## Constraints

Lombok generates code during compilation; the build, the generated bytecode (or `delombok` output), and the IDE's Lombok support are the sources of truth.

- **MANDATORY**: Compile after every change (`./gradlew compileJava` / `mvn compile`). Lombok validation errors appear during annotation processing.
- **VERIFY**: Run the project's tests after changes; report failures honestly with the compiler/test output; do not claim success until the build and tests pass.
- **PROCESSOR ON THE PATH**: Lombok must be a compile-only dependency **and** a registered annotation processor (Maven `provided`; Gradle `compileOnly` + `annotationProcessor`). A missing processor causes "cannot find symbol getX()/builder()" everywhere.
- **READ THE GENERATED CODE**: Use `delombok` (or the IDE's decompile) to confirm exactly what Lombok produced — never guess.
- **VERSION AWARENESS**: Gate newer annotations on the project's Lombok version (e.g. `@Locked` needs ≥ 1.18.32). Check the dependency version when unsure.
- **DON'T HAND-EDIT GENERATED CODE**: It's regenerated each build. Change the annotation, its attributes, a hand-written member (Lombok skips members that already exist), or `lombok.config`.
- **ENTITY & INHERITANCE TRAPS**: `@Data`/`@EqualsAndHashCode`/`@ToString` on JPA entities and subclasses are the top foot-guns (lazy-loading, bidirectional `StackOverflowError`, broken identity, missing `callSuper`). Tune with `onlyExplicitlyIncluded`/`@Exclude`/`callSuper`.
- **WITH MAPSTRUCT**: When both are annotation processors (this project uses both), add `org.projectlombok:lombok-mapstruct-binding` and order processors `mapstruct → lombok → binding`. See the companion `mapstruct` skill (§14.2).
- **BEFORE APPLYING**: Read the relevant chapter section in the reference for the attributes, the generated output, and the pitfall.

## When to use this skill

- Add or refactor Lombok annotations on a class (getters/setters, constructors, `@Data`/`@Value`, builders, logging)
- Diagnose "Lombok isn't generating code" / "cannot find symbol getX()/builder()" / phantom IDE errors
- Build an immutable type (`@Value`, `@Builder`, `@With`, `@SuperBuilder`) or a Jackson-deserializable builder (`@Jacksonized`)
- Fix `equals`/`hashCode`/`toString` problems — especially on JPA entities or class hierarchies
- Add a logger (`@Slf4j` etc.), safe locking (`@Synchronized`/`@Locked`), or null-checks (`@NonNull`)
- Set up Lombok in Maven/Gradle/IDE, configure `lombok.config`, or make Lombok coexist with MapStruct
- Understand or reproduce what an annotation generates (delombok), or migrate off Lombok

## Workflow

1. **Confirm the version and that the project compiles**

   Determine the Lombok version (dependency / `gradlew dependencies` / `pom.xml`). Run the build and stop if it does not compile. Gate every recommendation on whether the feature exists in that version (see the reference's since-version table).

2. **Read the reference and analyze the class**

   Read `references/lombok.md`. Map each requirement or smell to a specific practice by chapter section and title (e.g. "4.3 Make an immutable value type with `@Value`", "3.3 …set `callSuper` correctly"). Inspect the class shape and, where helpful, the delombok output.

3. **Apply the idiomatic Lombok solution**

   Prefer the smallest annotation that expresses intent; override one generated member by hand or via attributes/`lombok.config` rather than fighting the output. Watch the entity/inheritance pitfalls. Cite the practice you applied by chapter section and title.

4. **Recompile, inspect generated code, and verify**

   Recompile after each change, delombok or decompile to confirm the generated members, and run tests. Resolve warnings (missing `@Builder.Default`, missing `callSuper`) rather than ignoring them. Re-verify before declaring done.
